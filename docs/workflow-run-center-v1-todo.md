# Workflow Run Center V1 待做清单

## 文档目的

本文将 [Workflow 后续改进路线](./workflow-improvement-roadmap.md) 中的 Workflow Run Center 进一步拆解为可执行的开发清单。

V1 的目标是让用户能够从同一个 Workflow 查看任意历史 Run，理解一次运行的配置、节点阶段、耗时、错误和产物。V1 暂不实现失败节点重跑和完整成本分析，但数据合同需要为后续能力预留空间。

## 当前已具备能力

- Workflow 详情中已有 Runs 入口和运行历史抽屉。
- 可以按 Workflow 查看历史 Run，并选择某次 Run 查看详情。
- 已展示 Run 状态、开始时间、耗时和 Graph Version。
- 已展示冻结计划、节点事件、节点错误和节点消息历史。
- 节点详情已支持部分 Runtime、Channel、模型、attempt、Token、成本和耗时信息。
- Run、节点进度、事件和节点会话已经具备持久化基础。

## V1 待做清单

### 1. Run 查询与列表

- [x] 展示完整 Run 列表，不受当前运行状态限制。
- [x] 支持按状态筛选：运行中、等待处理、已完成、失败、已停止。
- [x] 支持按触发来源筛选：手动、定时、MCP、恢复、重跑。
- [x] 支持按时间范围筛选。
- [x] 支持按 Revision / Graph Version 筛选。
- [x] 列表展示开始时间、结束时间、总耗时、触发来源和最终结果。
- [x] 明确列表排序规则，并在重载后保持稳定。
- [x] 为空、加载中、加载失败和历史记录较多时提供明确状态；大列表按批次加载。

### 2. Run 观测合同

- [x] 为 Run 增加稳定的触发来源字段。
- [x] 为 Run 增加 Revision / Graph Version 快照字段。
- [x] 持久化 Runtime、Channel、模型及相关配置快照。
- [x] 配置快照只保留可解释的路由字段，不写入密钥、授权头、完整环境变量和敏感本地路径。
- [x] 为 Run 和节点事件增加稳定的 sequence。
- [x] 支持重载、恢复和重复订阅时按 sequence 去重。
- [x] 为节点结果增加输出摘要或 digest。
- [x] 明确旧数据缺少新增字段时的兼容读取策略。

### 3. Run 详情与只读快照

- [x] 详情展示 Run 的开始时间、结束时间、总耗时和最终状态。
- [x] 详情展示触发来源、Revision、Graph Version 和完整运行配置快照。
- [x] 将历史 Run 对应的画布切换为只读执行快照。
- [x] 历史详情使用 Run 快照、进度和事件数据展示，不提供修改历史记录的入口。
- [x] V1 不提供从历史 Run 直接发起后续操作的入口；未来新增操作必须创建新的可审计记录，不能修改历史 Run。
- [x] 支持在桌面端重启后继续查看已持久化的 Run 详情。

### 4. 节点时间线

- [x] 展示节点排队、开始、暂停、恢复、等待输入、等待审批、重试和完成阶段。
- [x] 区分实际执行耗时、等待用户耗时和等待审批耗时。
- [x] 根据持久化事件的时间和 sequence 记录每个阶段的开始和结束时间。
- [x] 并行节点在时间线上展示实际重叠区间。
- [x] 展示每个节点的 attempt 历史，而不是只显示最后一次尝试。
- [x] 节点状态颜色不能作为唯一信息表达方式，已同时显示文字和状态图标。

### 5. 节点详情

- [x] 展示节点输入摘要。
- [x] 展示节点输出摘要和 digest。
- [x] 展示错误详情、稳定错误码和用户可读信息。
- [x] 展示人工输入、审批和 Reviewer 决策。
- [x] 展示节点产物引用及安全预览信息。
- [x] 展示节点实际使用的 Runtime、Channel、模型、attempt、Token、成本和耗时。
- [x] 节点不存在观测数据时，明确显示“未开始”或“暂无数据”，不能伪造默认值。

### 6. 验证与测试

- [x] 测试用户可以从同一个 Workflow 打开任意历史 Run。
- [x] 测试应用重启后 Run 列表和详情仍然完整。
- [x] 测试筛选条件组合和稳定排序。
- [x] 测试并行节点时间区间存在重叠时的展示结果。
- [x] 测试执行、等待输入和等待审批耗时能够分别统计。
- [x] 测试事件重复订阅不会产生重复记录。
- [x] 测试旧版本持久化数据缺少新增观测字段时仍可读取。
- [x] 测试配置快照不会泄漏密钥、授权信息、环境变量和敏感路径。
- [x] 测试大量 Run 下列表按批次加载仍可使用。

## V1 暂不包含

- [ ] 失败节点重跑。
- [ ] 两个 Run 的拓扑、节点结果、耗时和成本对比。
- [ ] 两个 Revision 的质量对比。
- [ ] 完整成本趋势和 Evaluation 反馈闭环。
- [ ] MCP 对 Run 完整生命周期的控制工具。

这些能力应在 V1 的观测合同稳定后继续实现，不应通过临时字段或直接修改历史 Run 来提前实现。

## V1 验收标准

- [x] 用户可以从同一个 Workflow 打开任意历史 Run。
- [x] 用户可以按状态、触发来源、时间范围和 Revision 筛选 Run。
- [x] 用户可以查看某次 Run 的完整配置快照、节点时间线、错误和产物摘要。
- [x] 并行节点能够体现实际重叠区间。
- [x] 用户能够区分执行、等待输入和等待审批的耗时。
- [x] 应用重启后仍能查看上述信息。
- [x] 历史 Run 默认只读，查看历史不会触发新的执行。
- [x] 观测数据不包含密钥、授权头、完整环境变量或未经处理的敏感路径。

## 推荐实施顺序

1. 先扩展 Run、节点 attempt 和事件观测合同。
2. 再补齐持久化迁移、历史 Run 查询和筛选接口。
3. 实现只读 Run 列表、详情和节点时间线。
4. 增加配置快照、输入输出摘要、产物引用和敏感字段过滤。
5. 最后补齐恢复、重载、并行时间线和大数据量测试。
