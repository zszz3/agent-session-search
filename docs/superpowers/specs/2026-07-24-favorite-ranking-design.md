# 收藏会话继承置顶排序规则

## 目标

移除 Pin 功能后，由收藏状态承接原来的搜索排序价值。用户仍可通过一个明确、可管理的状态让重要会话优先展示。

## 排序规则

收藏会话继承原 Pin 的三项规则：

- 无关键词时，收藏会话排在未收藏会话之前；同组内继续使用当前选择的时间排序。
- 普通关键词相关性计算中，收藏会话额外加 25 分。
- 智能排序中，收藏会话的综合得分乘以 1.2。

收藏筛选、收藏和取消收藏操作保持不变。

## 数据兼容

不迁移历史 `pinned` 数据，也不把 `pinned = true` 的会话自动设为收藏。旧 `pinned` 字段继续作为数据库兼容列保留，但不参与展示、筛选或排序。

## 测试

- 直接给较旧会话设置收藏，验证无关键词列表优先返回该会话。
- 使用相同关键词构造收藏与未收藏会话，验证普通相关性增加 25 分。
- 构造相同相关性和时间条件，验证智能排序应用 1.2 倍收藏权重。
- 保留旧 `pinned` 值不影响排序的回归测试。
