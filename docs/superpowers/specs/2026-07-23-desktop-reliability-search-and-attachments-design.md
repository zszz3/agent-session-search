# 桌面可靠性、快速搜索与会话附件设计

## 背景与目标

本次改动继续提交到现有 Cursor Remote 会话修复分支和 MR，不创建新分支。除保留已完成的 Cursor Remote SSH 归类与空会话过滤外，补齐以下用户能力：

- README 直接提供国内 npm 与 Electron 镜像安装命令。
- Codex 登录后能够及时显示额度，偶发网络超时不会清空上次可用数据。
- App 和 CLI 更新过程展示下载与安装进度，失败时保留旧版本。
- macOS 菜单栏新增轻量会话快速搜索，不重做已有打开、刷新和退出菜单。
- macOS 可控制是否在程序坞中显示，默认显示。
- 本地会话附件可预览或打开；远程会话同步开启后，安全识别出的附件默认随会话上传。

Node 低版本兼容不在本次范围内。正式安装后的启动继续使用包内 Electron 和现有稳定 Node 更新机制。

## 方案比较

### 采用：同一 MR 内按服务边界垂直实现

额度、更新、附件和快速搜索分别拥有独立的数据模型或生命周期服务，通过类型化 IPC 暴露给渲染层。设置沿用 `AppSettings` 与 `electron-store`，界面复用现有样式和会话搜索能力。每个能力独立测试，最终更新同一份 release note。

这既满足继续使用现有 MR 的要求，也避免把网络重试、文件读取、进程替换和窗口生命周期继续堆入 `src/main/index.ts` 或 `App.tsx`。

### 不采用：直接在现有组件和脚本中追加状态

改动文件较少，但额度缓存、更新进程、附件下载和多个窗口会共享隐式状态，容易出现一次网络失败覆盖成功数据、窗口关闭后丢进度、远程附件路径被误用等问题。

### 不采用：拆成多个分支和 MR

附件和更新本可独立发布，但用户已明确要求继续更新现有 MR。本次仍在代码边界和提交顺序上保持可审查性，不另建开发分支。

## 用户设置

新增两个设置：

- `showInDock: boolean`：默认 `true`，仅在 macOS 设置界面展示。
- `syncSessionAttachments: boolean`：默认 `true`。只有远程会话同步已启用时才生效；关闭后仍同步会话文字，但不上传新的附件。

旧设置文件缺少字段时通过 `defaultSettings` 自动补齐。切换 `showInDock` 后立即调用 macOS Dock API，重启后保持。关闭 Dock 显示不会销毁菜单栏图标或主窗口恢复入口。

## Codex 额度可靠性

### 服务边界

新增主进程额度服务，负责：

- 读取当前 Codex OAuth 身份并生成不可逆账号指纹，不持久化 Token。
- 调用现有 `loadUsageQuotaSnapshot`，合并重试、缓存和登录文件监听。
- 向 renderer 发布额度更新事件，保留现有手动刷新接口。
- 将安全的额度值、账号指纹和成功时间缓存到应用数据目录。

渲染层不再自行决定一次错误是否覆盖数据，只渲染服务返回的 `fresh`、`stale`、`auth-required` 或 `unavailable` 状态。

### 请求与缓存规则

- 网络超时、DNS、连接重置等瞬时错误最多重试两次，使用短退避并限制整体请求时间。
- HTTP 401/403 不重试，提示重新登录；HTTP 429 和其他确定性响应不做盲目重试。
- 成功响应立即更新内存和磁盘缓存。
- 同账号瞬时失败时继续展示上次成功数据，并显示最后更新时间和失败原因。
- 缓存超过 24 小时不再作为可用额度展示；已过重置时间的窗口明确标记过期。
- 账号指纹变化后不得展示旧账号缓存。

### 登录变化

监听当前 `auth.json` 的创建、替换和修改。事件防抖后重新读取；遇到登录程序原子替换或短暂半写入状态时延迟重试。登录、退出或切换账号均触发主动刷新，无需重启 AgentRecall。

## 完整更新进度

### 进度模型

更新状态扩展为以下阶段：

1. `checking`
2. `downloading`
3. `verifying`
4. `staging`
5. `validating`
6. `restarting`
7. `completed`
8. `error`

下载阶段提供 `downloadedBytes`、`totalBytes`、百分比和速度。npm 安装和 Electron 校验没有可靠总字节数，只显示真实阶段、持续时间和最近的安全日志摘要，不伪造百分比。

App 通过现有更新 IPC 接收状态事件；CLI 使用同一进度模型渲染文本进度条和阶段信息。

### 暂存安装

现有流程在 App 退出后才开始下载，无法持续显示进度。本次改为：

1. App 创建专用更新进度窗口并隐藏普通主窗口。
2. 将 Release 包流式下载到临时目录，边写磁盘边计算 SHA-256，避免把整个包读入内存。
3. 使用稳定 Node 和 npm 把新版本安装到与真实全局包隔离的暂存目录，完成依赖与 Electron Runtime 下载。
4. 校验暂存包的 CLI、主入口和 Electron Runtime。
5. 进度窗口进入“正在重启”，启动现有独立 finalizer，然后退出当前 App。
6. finalizer 等待旧进程结束，备份旧包，把已校验的暂存包替换到原全局路径，并启动新版本。
7. 新版本确认安装结果并清理备份、锁和临时目录。

全局 CLI 启动脚本路径保持不变，因此包目录替换后继续指向新版本。暂存安装禁止写入正式集成配置；新版本启动后沿用现有修复逻辑刷新 MCP、Hooks 等集成。

### 失败与回滚

- 下载、校验或暂存失败时 App 不退出，进度窗口展示原因、重试入口和手动安装命令。
- finalizer 替换失败时恢复旧包并重新启动旧版本。
- 同一时间只允许一个更新，沿用当前安装锁。
- 更新日志过滤 Token、环境变量和本地敏感路径。
- 所有更新测试使用临时 HOME、npm prefix、合成 Release 包和合成 Electron Runtime。

## macOS 菜单栏快速搜索

现有菜单栏的打开、刷新和退出能力保持不变，仅增加“快速搜索会话…”。

选择后显示独立的轻量 BrowserWindow：

- 窗口靠近菜单栏出现，失去焦点或按 Esc 时隐藏而不是销毁。
- 搜索框自动聚焦，复用现有数据库搜索和排序规则。
- 最多展示 8 条结果，包含标题、Agent 来源、项目和相对时间。
- 上下方向键切换选择；回车隐藏快速窗口、显示主窗口并打开对应会话详情。
- 不复制索引，不在 renderer 直接读取数据库。
- Dock 隐藏时仍可完整使用。

快速搜索采用独立 renderer 入口或轻量模式，不启动主界面的统计、额度、Skills 与远程同步加载。

## macOS Dock 行为

- `showInDock=true` 时保持当前应用图标和菜单行为。
- `showInDock=false` 时调用 `app.dock.hide()`，菜单栏图标继续常驻。
- 重新开启时调用 `app.dock.show()`，无需重启。
- 关闭最后一个窗口不退出应用；菜单栏“退出”与 Cmd+Q 仍执行完整清理。
- 设置项仅在 `process.platform === "darwin"` 时显示，Windows 行为不变。

## 会话附件

### 附件定义与安全边界

只有原始会话格式中明确声明为图片或文件附件的内容才进入附件系统。以下内容不得自动读取或上传：

- 普通消息文字中出现的路径。
- 工具调用参数、工具读取记录或命令输出中的路径。
- 目录、符号链接、设备文件或无法确定大小的对象。
- 由不受支持的自定义字段猜测出的路径。

可自动同步的附件包括：

- 会话记录内嵌的 Base64 图片或文件。
- Agent 明确生成在其受控附件目录中的普通文件。
- 已由 AgentRecall 管理并通过哈希校验的附件缓存。

其他明确附件可以在本地显示元数据和原始路径，但在上传前标记为“不安全或不可读取”，不会因为默认开关开启而越权读取任意文件。

### 数据模型

新增 `SessionAttachment`：

- 稳定 ID、所属 `sessionKey` 和 `messageIndex`
- 原始顺序、显示名、类型、MIME、字节数
- SHA-256、可用状态和失败原因
- 本地托管对象引用或安全原始文件引用
- 远程对象键，不包含上传端绝对路径

SQLite 新增附件表，只保存元数据和托管对象引用，不在数据库存大 Blob。内嵌内容解码后写入应用管理的内容寻址缓存。重新索引和删除会话时清理不再被引用的缓存对象。

默认限制为单附件 25 MiB、单会话 100 MiB。超过限制的附件保留名称、类型和大小，明确显示未缓存或未上传原因。限制保护本地磁盘、内存和个人 Supabase 配额。

### 解析与本地查看

会话格式适配器返回文本消息和附件描述，支持 Codex `input_image`、Claude 图片块以及兼容来源的明确附件块。附件解析失败不得丢失同一条消息的文字。

详情页在对应消息下显示附件卡：

- 图片显示缩略图，点击打开应用内预览。
- PDF 使用应用内安全预览。
- UTF-8 文本和代码提供只读预览。
- 其他普通文件显示名称、类型和大小，可交给系统默认应用打开。

renderer 只传附件 ID。主进程校验 ID、会话归属、对象状态和实际文件类型后，通过受控协议提供预览或调用 `shell.openPath`，不接受 renderer 直接传入任意文件路径。

### 远程上传和查看

`syncSessionAttachments=true` 且会话上传或更新时：

1. 先根据附件 SHA-256 和大小构建 manifest。
2. 将安全可读附件上传到远程会话的私有 Storage 前缀。
3. 上传包含附件 manifest 的详情快照和 portable snapshot。
4. 最后更新远程会话行；任一步失败都删除本次新对象，不破坏上一版本。

附件变化纳入会话内容哈希。旧远程会话不会在升级后自动全量回传；再次上传、Hook 更新或冲突处理选择本地版本时补齐附件。

查看远程会话时只加载附件元数据。用户点击附件后才下载到应用缓存，校验 SHA-256 和大小后预览或打开。远程 manifest 不暴露上传设备的绝对路径。删除或替换远程会话时删除对应版本附件对象。

## README 镜像命令

默认 GitHub 安装命令后直接增加：

```bash
ELECTRON_MIRROR=https://npmmirror.com/mirrors/electron/ \
npm install -g https://github.com/zszz3/AgentRecall/releases/latest/download/agent-recall.tgz \
  --registry=https://registry.npmmirror.com
```

说明 Release 压缩包仍由 GitHub 下载，npm 依赖和 Electron Runtime 使用镜像；命令只影响本次安装，不改用户全局 npm 配置。

## IPC 与代码边界

- `QuotaService`：额度请求、缓存、身份监听和状态发布。
- `AppUpdateService`：更新状态机、暂存安装和进度发布；finalizer 只负责退出后的原子替换与回滚。
- `AttachmentService`：附件对象缓存、校验、预览、打开和远程对象生命周期。
- `QuickSearchWindowController`：窗口创建、隐藏、定位和选中结果交接。
- shared IPC 使用现有 Zod 合约；renderer 不接触 Token、文件系统路径、npm 命令或 Storage 凭据。

只在这些边界确实承担生命周期和安全职责时新增模块；单一调用点的纯映射逻辑继续留在所属函数内。

## 数据迁移与兼容

- SQLite migration 新增附件表和必要索引，不重写原始 Agent 会话。
- 现有消息、标签、收藏、自定义标题和远程同步绑定保持不变。
- 远程详情快照升级为兼容旧版的 schema；旧 snapshot 没有附件时按空数组处理。
- 附件对象复用现有私有 bucket 和对象策略，不要求用户执行新的远程表结构升级。
- macOS 新设置缺失时使用默认值；Windows 忽略 Dock 设置。

## 测试与验证

### 额度

- 瞬时超时重试后成功。
- 连续失败使用同账号有效缓存。
- 401/403 不重试并提示登录。
- 登录文件创建、原子替换、半写入和账号切换。
- 缓存过期、窗口重置过期和 Token 脱敏。

### 更新

- 流式下载的字节进度和未知 `Content-Length`。
- 校验失败不退出 App。
- 暂存 npm 安装、Electron 校验、finalizer 替换和旧包回滚。
- CLI 文本进度和 App IPC 进度使用同一状态。
- macOS、Windows 路径与全局启动脚本保持有效。

### 快速搜索与 Dock

- 菜单动作创建或复用单一快速窗口。
- 搜索数量上限、键盘选择、回车打开详情、失焦隐藏。
- Dock 默认显示、即时隐藏/恢复以及 Windows 无操作。

### 附件

- 各来源的内嵌图片和明确文件块解析。
- 普通路径、工具路径、符号链接和超限文件不被读取或上传。
- 本地缓存哈希、引用清理、图片/PDF/文本预览和系统打开。
- 远程 manifest、默认上传、关闭开关、按需下载、哈希校验、失败回滚和删除清理。
- 旧远程 snapshot 兼容。

所有测试使用临时 HOME、临时 npm prefix、临时数据库、合成会话与合成附件。不得读取、上传、修改或删除开发者真实的 Codex、Claude、Cursor、Supabase、Electron 或会话数据。完成后执行 `npm test`、`npm run build`、`npm run release-note:check`、包安装冒烟测试和 `git diff --check`，并确保没有残留 UI/更新进程或临时文件。
