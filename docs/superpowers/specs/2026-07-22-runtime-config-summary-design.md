# Runtime 配置摘要式页面设计

## 背景与目标

Runtime 页当前把 Runtime、配置、余额、Provider 全量列表、插件和模型以相近的视觉权重连续铺开。Provider 实际属于低频切换操作，却占据页面最大面积，导致首屏难以回答用户最关心的三个问题：当前使用什么配置、配置是否可用、下一步可以做什么。

本次改版只调整 Runtime 的信息架构和交互呈现，不改变 Runtime、Provider、余额、测试、导入、插件或模型的数据逻辑。

## 页面结构

页面正文使用居中的有限宽度内容区，避免表单横向拉满。内容从上到下分为三层：

1. 紧凑 Runtime 切换栏：保留 Codex、Claude Code、API、Hermes、OpenCode、OpenClaw，但缩短高度并强化当前项。
2. 当前配置摘要：合并配置选择、当前 Provider、余额、测试状态和主要操作。摘要以当前 Runtime 的强调色作为细窄识别线，其他区域保持现有中性色。
3. 按需配置区：模型、Codex 插件和高级 Provider 配置分别作为默认折叠的独立区段，仅展示名称和数量摘要。

摘要区默认呈现：

- Runtime 与配置名称，例如 `Codex · Codex OpenAI`。
- 当前 Provider，例如 `OpenAI Official`。
- 最近一次余额结果或“尚未查询”。
- 测试、导入本地配置、刷新余额、新增配置和删除配置操作。
- 测试或保存产生的状态信息，紧邻摘要展示，不占用新的整宽卡片。

页面顶部继续保留统一的 Runtime 标题栏和保存按钮。

## Provider 选择

默认页面不再渲染完整 Provider 按钮墙，只显示当前 Provider 和“更换 Provider”按钮。

点击后打开模态选择器：

- 顶部提供关键词搜索。
- Provider 仍按本地、官方、国内官方、云服务商、聚合服务、第三方、自定义分类。
- 当前 Provider 有明确选中状态。
- 选择后立即沿用现有预设应用逻辑并关闭选择器。
- API Key 和可配置 Model ID 只在当前 Provider 需要时显示在摘要下方的连接设置中。
- Escape、关闭按钮和点击遮罩均可退出，键盘焦点保持可用。

Provider 选择器作为独立组件承载弹层生命周期、搜索和分类列表；其余仅有一个调用点的页面逻辑继续保留在 `RuntimePage`，避免增加无意义包装层。

## 折叠区段

- 模型：标题显示模型数量，展开后沿用现有刷新、添加、编辑和删除能力。
- Codex 插件：仅 Codex 显示，标题显示已配置数量；展开后沿用目录加载、手动添加、启用和删除能力。
- 高级配置：保持默认折叠，包含底层渠道、环境变量、Header 等低频字段。

折叠状态仅属于当前页面会话，不新增持久化字段。切换 Runtime 或配置时仍以折叠状态呈现，避免页面突然变长。

## 状态与异常

- 测试中、测试成功和测试失败继续显示现有信息，但压缩到摘要下方；失败详情允许展开查看。
- 余额不可用、缺少 Key 或查询失败时，摘要显示简短原因，刷新操作仍可重试。
- 没有配置时保留现有导入或新建入口，不展示空的 Provider、插件和模型区段。
- Provider 搜索无结果时说明“没有匹配的 Provider”，不显示空白弹层。

## 响应式与视觉约束

- 宽窗口下内容区使用 `1040px` 最大宽度并居中；摘要内部使用自适应列，不制造整页空卡片。
- 窄窗口下摘要改为纵向排列，操作区可换行，Runtime 切换栏允许横向滚动。
- 继续使用现有 Geist 字体、主题变量、圆角和按钮体系，不新增独立配色系统。
- 动效仅用于弹层出现和折叠展开，并遵守 `prefers-reduced-motion`。

## 验证

- 组件测试覆盖默认不显示 Provider 全量列表、打开选择器后可搜索和选择、当前 Provider 摘要正确。
- 保留并更新现有 Runtime 页面结构测试，确保导入、测试、余额、新增、删除、模型和插件入口仍存在。
- 执行完整测试、生产构建和 release note 校验。
- 在实际 Electron 窗口检查首屏密度、窄窗口换行、Provider 弹层及各折叠区段。

## 非目标

- 不调整 Runtime 或 Provider 的数据结构。
- 不改变 API Key 保存方式、模型目录请求或余额查询逻辑。
- 不新增 Provider 排序、收藏或最近使用记录。
- 不改 Agent、MCP、Workflow 等其他页面。
