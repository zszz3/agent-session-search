# 工作台用量周期选择器设计

## 背景

工作台用量卡当前把“今天、7 天、30 天、全部”作为分段按钮放在四个核心指标左侧。控件没有所属标题，又与统计数字处于同一行，视觉上像一组悬空导航，也压缩了指标的横向空间。

本次只调整用量周期控件的呈现和位置，不改变统计口径、默认周期、刷新行为或数据请求。

## 已选方案

使用紧凑的单选下拉框替代四个常驻分段按钮：

- 用量卡增加明确的“用量”标题。
- 当前周期以下拉框形式放在卡片右上角，刷新按钮紧邻其右侧。
- 下拉选项仍为“今天、7 天、30 天、全部”，当前值使用现有本地化文案。
- 四个核心指标独占标题栏下方的一整行，不再与周期控件争抢宽度。
- Token 构成和来源统计继续位于指标下方，布局和数据保持不变。

## 交互与状态

- 点击下拉框选择周期后，继续调用现有周期切换回调并刷新对应统计。
- 选择完成后，下拉框只显示当前周期；键盘操作和焦点样式遵循原生单选控件行为。
- 统计刷新期间，刷新按钮维持现有禁用和旋转反馈；周期下拉框仍显示当前选择。
- 窄窗口不展开为常驻四项按钮，避免再次挤压指标。

## 视觉约束

- 下拉框与工作台现有筛选控件使用同一高度、圆角、边框和字体层级。
- “用量”标题承担信息归属，不增加说明性副标题，保持顶部总览紧凑。
- 周期和刷新组成一个右对齐操作组；它们不与数字基线对齐。
- 不增加卡片整体高度，不改变模型额度卡和预留模块的比例。

## 验收标准

- 工作台不再常驻显示四个并排周期按钮，右上角显示当前周期下拉框。
- 下拉框包含四个原有周期，选择任意周期后统计结果与原逻辑一致。
- 会话、消息、Token 和缓存率在同一完整横行中对齐。
- 刷新、加载反馈、中英文文案和窄窗口布局保持可用。
- 现有工作台测试通过，并增加周期下拉结构与选项的界面契约测试。
