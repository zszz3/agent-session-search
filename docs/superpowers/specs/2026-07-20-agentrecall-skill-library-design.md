# AgentRecall 托管 Skill 库设计

## 目标

将 Skills 页面从“扫描多个 Agent 目录并直接管理副本”调整为以 AgentRecall 托管库为核心的管理界面。页面主体沿用 Multi Agent Chat 的双栏信息架构：左侧选择已加入 AgentRecall 的 Skill，右侧阅读内容并管理安装、同步与版本。`skills.sh` 只作为“发现 Skill”的次级入口，不常驻主页面。

## 已确认的产品规则

- AgentRecall 托管库是 Skill 内容的唯一可信源。
- Codex、Claude Code 和 Trae 是安装目标，不是相互独立的内容源。
- macOS/Linux 使用目录符号链接，Windows 使用目录 Junction，将托管副本安装到目标目录。
- 当前机器上已经存在的 Skill 不会被自动复制、移动、覆盖或接管。用户通过“导入本机 Skill”明确选择后，才会把副本加入托管库。
- 云端同步只处理托管库中的 Skill。目标目录中的链接不重复上传。
- `skills.sh` 榜单通过“发现 Skill”按钮进入；“加入 Skill 库”只导入托管库，不自动部署到任何 Agent。
- 同名目录、普通目录、非 AgentRecall 创建的链接和内容冲突都必须停止操作并给出可执行的提示。

## 页面结构

### 主页面

Skills 页面保留应用统一标题栏，右上角提供：

- 导入本机 Skill
- 发现 Skill
- 刷新

标题栏下方采用双栏布局：

- 左栏：托管 Skill 搜索、分类筛选和列表。列表显示名称、描述、来源、使用次数及紧凑的安装状态。
- 右栏：当前 Skill 的说明与 Markdown 内容。顶部展示来源和状态；主要操作是选择 Codex、Claude Code、Trae 的安装目标；云端同步、版本历史和 Diff 放在次级区域。

托管库为空时，页面明确提供“导入本机 Skill”和“发现 Skill”两个起点。

### 导入本机 Skill

导入浮层扫描现有受支持 Skill 根目录，排除已经由 AgentRecall 链接安装的项目，并按 Agent 与位置展示候选。用户可以单选或批量选择。

导入时复制完整 Skill 目录到托管库，保留附属脚本、引用和资源，不修改原目录。若托管库已有同名 Skill，则先比较内容；完全一致时复用，内容不同时报告冲突并跳过。

### 发现 Skill

“发现 Skill”打开覆盖工作区的大尺寸浮层：

- 顶部提供搜索与关闭操作。
- 左侧使用 `skills.sh` all-time 数据按安装量展示榜单，支持分页加载。
- 右侧展示名称、仓库、安装量、来源链接和完整 Skill 内容。
- 主操作为“加入 Skill 库”。成功后关闭浮层，刷新主列表并选中新加入的 Skill。

榜单元数据使用短时本地缓存；Skill 内容仅在打开详情或执行导入时按需下载。网络失败时保留已缓存数据并提供重试，不影响本地 Skill 管理。

## 托管数据与文件布局

托管根目录位于应用用户数据目录下，并按稳定的 Skill 标识分目录保存完整文件树。每个条目包含应用生成的元数据，记录：

- 稳定标识与显示名称
- 原始来源类型、路径或 URL
- 导入时间与内容指纹
- 可选分类

展示所需的 Markdown、描述和资源从托管目录读取。安装状态通过检查目标路径是否为指向该托管目录的受控链接得出，不以易失的前端状态为准。

## 安装与卸载

用户在详情中勾选安装目标并应用更改：

- 目标不存在时创建受控链接。
- 目标已经是指向当前托管目录的链接时视为已安装。
- 目标是普通文件、普通目录或指向其他位置的链接时拒绝覆盖。
- 卸载只删除确认由 AgentRecall 创建且仍指向当前托管目录的链接。
- 删除托管 Skill 前，先提示仍连接的目标；确认后先安全移除受控链接，再删除托管副本。原始导入来源不受影响。

## 云端同步、版本与 Diff

现有同步能力改为枚举托管根目录，而不是把多个 Agent 目录中的副本当作独立 Skill。远端身份由托管 Skill 的稳定标识和内容指纹确定。

右侧详情保留：

- 上传或更新云端副本
- 安装远端 Skill 到本机托管库
- 版本选择
- 本地与远端 Diff

批量上传和下载仍可用，但收进同步区域，不再占满主工具栏。旧远端记录继续兼容展示；下载后进入托管库，再由用户决定安装目标。

## 使用次数

使用次数继续来自现有 Hook 和会话事件。统计时将 Agent 目录中的受控链接解析回托管 Skill，多个 Agent 使用同一 Skill 时汇总到同一个条目；未导入的外部 Skill 不进入主库统计。

## 安全与跨平台

- 所有导入、安装和测试使用显式目录边界与路径校验，拒绝路径穿越。
- 不读取或上传 Skill 内容以外的用户数据。
- 测试必须使用临时 HOME、临时应用数据目录和合成 Skill，不触碰真实 Codex、Claude Code、Trae 或同步数据。
- macOS/Linux 验证符号链接；Windows 路径逻辑使用 Junction，并由平台分支测试覆盖。
- `skills.sh` 请求不携带本地凭证、会话内容或机器信息。

## 视觉方向

整体延续 AgentRecall 当前紧凑工作台语言，不引入独立网站式视觉。左侧列表强调高密度浏览，右侧采用安静的文档阅读区；唯一突出元素是横向安装目标轨道，让用户一眼看出一个 Skill 已连接到哪些 Agent。发现浮层沿用同一双栏骨架，榜单排名和安装量只承担信息结构，不作为装饰性大数字。

## 验证范围

- 托管库创建、枚举、导入、去重、冲突与删除。
- macOS/Linux 链接及 Windows Junction 的安全安装与卸载判断。
- 已安装状态与使用次数归并。
- `skills.sh` 榜单解析、分页、缓存、详情下载和失败降级。
- 主页面、导入浮层、发现浮层、键盘访问、中英文与空状态。
- 现有同步、版本、Diff 和批量操作迁移后的回归测试。
- 全量单元测试、类型检查、构建与 release note 校验。
