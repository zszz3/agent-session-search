# Runtime 配置选择器精简设计

## 目标

让 Runtime 页面只展示一次运行时层级，避免默认配置被呈现为重复的 `Codex · Codex OpenAI · OpenAI`。用户应先选择执行器，再管理该执行器下的配置。

## 页面结构

- Codex、Claude Code、API、Hermes、OpenCode、OpenClaw 继续作为执行器 Tab，保持现有选择逻辑。
- 移除编辑区上方跨所有执行器的独立配置工具栏。
- 在编辑区标题行放置当前执行器的配置下拉框，以及新增、删除配置入口。
- 下拉框只列出当前执行器的配置，因此选项只显示配置名；Provider 信息由下方表单展示，不在选择器中重复。
- 单个默认配置仍显示下拉框，保证新增第二套配置后无需改变页面结构。

## 交互与数据

- 切换执行器时，沿用现有规则选中该执行器最近或默认的配置。
- 切换配置只改变当前执行器内的配置，不跨 Runtime 跳转。
- 新增配置归属于当前执行器，并立即成为当前配置。
- 删除规则、被 Agent 引用时的保护、保存、测试、导入本地配置和余额查询逻辑保持不变。
- 配置没有自定义名称时继续回退到配置 ID，避免空白选项。

## 视觉约束

- 编辑区标题保持一行紧凑控制轨：Runtime 徽标、配置下拉、新增、删除位于左侧，导入和测试位于右侧。
- 小窗口允许控制轨在容器内换行，按钮和下拉框不得溢出。
- 沿用 AgentRecall 现有颜色、字号、圆角和焦点样式，不引入新的装饰层级。

## 验证

- 组件测试验证全局配置工具栏消失，当前 Runtime 的配置下拉仍可用。
- 测试验证下拉选项不再拼接 Runtime 和 Provider 名称，并且不展示其他 Runtime 的配置。
- 运行类型检查、完整测试、构建和 release note 检查。
