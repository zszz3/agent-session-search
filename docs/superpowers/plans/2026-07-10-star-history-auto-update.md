# Star History Auto Update Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Show a live GitHub Star count at the top of both READMEs and regenerate the repository's Star History SVG automatically when its Stargazer history changes.

**Architecture:** A dependency-free Node 22 script fetches authenticated Stargazer timestamps from the GitHub REST API, converts them into a deterministic daily cumulative series, and renders `assets/star-history.svg`. A scheduled GitHub Actions workflow runs the script daily and commits only when the generated SVG differs. README badges use Shields.io directly and therefore update independently of the scheduled chart.

**Tech Stack:** Node.js 22 ESM, `node:test`, GitHub REST API, GitHub Actions, SVG, Shields.io
